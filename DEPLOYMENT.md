# Render Deployment Guide

## Architecture
```
Browser ‚îÄ‚îÄ‚ñ∫ cmms-frontend (Next.js)  ‚îÄ‚îÄ‚ñ∫ /api/* rewrites ‚îÄ‚îÄ‚ñ∫ cmms-backend (FastAPI)
                                                                      ‚îÇ
                                                            Supabase PostgreSQL
```
All API calls from the browser go through Next.js server-side rewrites (no direct CORS exposure).

---

## Pre-requisites
- Render account at https://render.com
- Supabase project (free tier works) with migrations applied
- Resend account + API key for email (https://resend.com)
- Git repository pushed to GitHub/GitLab

---

## Deploy via Blueprint (render.yaml) ‚Äî Recommended

1. Go to **Render Dashboard** ‚Üí **Blueprints** ‚Üí **New Blueprint Instance**
2. Connect your repo and select the root directory
3. Render will detect `render.yaml` and create both services automatically
4. Fill in the `sync: false` environment variables (see tables below)
5. Click **Apply** ‚Äî backend deploys first, frontend follows

---

## Environment Variables

### Backend (`cmms-backend`)

| Variable | Value | Where to get it |
|---|---|---|
| `SUPABASE_URL` | `https://<ref>.supabase.co` | Supabase ‚Üí Project Settings ‚Üí API |
| `SUPABASE_KEY` | anon/public key | Supabase ‚Üí Project Settings ‚Üí API |
| `SUPABASE_SERVICE_KEY` | service_role key | Supabase ‚Üí Project Settings ‚Üí API |
| `DATABASE_URL` | `postgresql://postgres:<pw>@db.<ref>.supabase.co:5432/postgres` | Supabase ‚Üí Project Settings ‚Üí Database |
| `JWT_SECRET_KEY` | auto-generated by Render | ‚Äî |
| `RESEND_API_KEY` | `re_xxxxx` | https://resend.com ‚Üí API Keys |
| `EMAIL_FROM_ADDRESS` | `noreply@cmms.utm.my` | pre-filled |
| `EMAIL_FROM_NAME` | `CMMS` | pre-filled |
| `ENVIRONMENT` | `production` | pre-filled |
| `CORS_ORIGINS` | `https://cmms-frontend.onrender.com` | Set **after** frontend deploys |

### Frontend (`cmms-frontend`)

| Variable | Value | Where to get it |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | auto-wired from `cmms-backend` URL | Set by Render blueprint |
| `NEXT_PUBLIC_SUPABASE_URL` | `https://<ref>.supabase.co` | Supabase ‚Üí Project Settings ‚Üí API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon/public key | Supabase ‚Üí Project Settings ‚Üí API |
| `NODE_ENV` | `production` | pre-filled |

## Manual Deploy (without Blueprint)

### 1 ‚Äî Backend

1. **New Web Service** ‚Üí connect repo ‚Üí root dir: `.`
2. Runtime: **Python 3**
3. Build command: `pip install -r backend/requirements.txt`
4. Start command: `uvicorn backend.main:app --host 0.0.0.0 --port $PORT`
5. Add all backend env vars from the table above

### 2 ‚Äî Frontend

1. **New Web Service** ‚Üí connect repo ‚Üí root dir: `frontend`
2. Runtime: **Node**
3. Build command: `npm install && npm run build`
4. Start command: `npm start`
5. Add all frontend env vars (use the backend service URL for `NEXT_PUBLIC_API_URL`)

### 3 ‚Äî Post-deploy CORS fix

After both services are live, copy the frontend URL (e.g. `https://cmms-frontend.onrender.com`)
and set it as `CORS_ORIGINS` in the **backend** environment variables, then redeploy the backend.

## Database Migrations

Run migrations against Supabase before the first deploy:

```bash
# From project root ‚Äî requires psql or Supabase CLI
psql "$DATABASE_URL" -f supabase/migrations/001_initial_schema.sql
psql "$DATABASE_URL" -f supabase/migrations/002_sprint2_schema.sql
```

Or use the Supabase Dashboard ‚Üí SQL Editor.

## Troubleshooting

| Symptom | Fix |
|---|---|
| Frontend shows blank page | Check `NEXT_PUBLIC_API_URL` is set and backend is running |
| 500 on login | Check `SUPABASE_SERVICE_KEY` is set (not just anon key) |
| Emails not sending | Verify `RESEND_API_KEY` and that `noreply@cmms.utm.my` domain is verified in Resend |
| CORS errors in browser | Usually means direct backend calls ‚Äî should not happen with Next.js rewrite proxy |
